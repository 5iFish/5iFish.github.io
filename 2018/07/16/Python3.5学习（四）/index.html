一、装饰器

1. 定义

本质是函数，（装饰其他函数）就是为其他函数添加附加功能

'''
#例：计算程序运行消耗的时间
import  time
def timmer(func):
    def warpper(*args,**kwargs):
        start_time = time.time()
        func()
        stop_time=time.time()
        print('the func run time is %s'%(stop_time-start_time))
    return warpper
@timmer
def test1():
    time.sleep(3)
    print('in the test1')
test1()
'''

2. 原则

  1. 不能修改被装饰的函数的源代码
  2. 函数调用方式不能修改

3. 储备知识

  1. 函数即“变量”

  1. 高阶函数
a:把一个函数名当做实参传给另外一个函数（在不修改被装饰函数源代码的情况下添加新的功能）

b:返回值中包含函数名
  1. 嵌套函数

高阶函数+嵌套函数=>装饰器

4. 实现装饰器过程1

仅仅使用高阶函数来实现例子：

import time
def bar():
    time.sleep(3)
    print("in the bar")
def test1(func):
    start_time= time.time()
    func()  #run bar
    stop_time = time.time()
    print("the func run time is %s"%(stop_time-start_time))
#改变了源程序中的调用函数
test1(bar)

假如将最后一行的函数调用改为：

bar = test1(bar)
bar()

可以发现抛开bar = test1(bar)我们没有更改原函数

5. 实现装饰器过程2

当我们采用有返回值调用修饰函数时，代码可以写成：

def deco(func):
    star_time=time.time()
    func()
    stop_time = time.time()
    print("the func run time is %s"%(stop_time-star_time))
    return func
#采用调用方式为：
test1=deco(test1)
test1()

6. 实现装饰器过程3

我们发现运行结果中运行了两次func()，分析调用过程，我们发现deco(test1)中运行了一次func，test1又运行一次，我们要保证只调用一次源代码，因此要将deco中不运行func，因此我们可以使用嵌套函数：

def timer(func):
    def deco():
        star_time=time.time()
        func()
        stop_time = time.time()
        print("the func run time is %s"%(stop_time-star_time))
    return deco
def test1():
#我们发现在执行timer(test1)时，嵌套函数中只是定义了deco函数，并未发现deco()，只是返回了一个创建的局部函数的内存地址
#因此在调用时，我们应该采用：
test1=timer(test1)
test1()
#程序执行顺序为先为timer创建空间计时器，然后为test1创建空间原功能区
#接下来执行timer(test1),计时器的空间地址f赋值给func,执行def deco()创建空间为新功能区
#return deco,返回为新功能区的地址，赋值给test1，接下来调用test1()
#由于test1指向新功能区，因此test1()执行新功能区的功能
#其中func由于指向原功能区，因此func()为执行原功能区功能
#最后将test1=timer(test1)替换为@timer写在需要添加装饰的函数前面
#至此实现了装饰器的功能

※重点理解：程序在运行时会从头一直到执行代码，为执行代码之前的函数、变量创建内存地址

7. 通用装饰器

当执行装饰器的时候，有时候原函数有传递参数，因此多使用参数不固定的形式：

def timer(func):
    def deco(*args,**kwargs):#通用装饰器
        star_time=time.time()
        func(*args,**kwargs)
        stop_time = time.time()
        print("the func run time is %s"%(stop_time-star_time))
    return deco

@timer
def test1():
    time.sleep(3)
    print('in the test1')
@timer
def test2(name,*args):#参数不固定
    time.sleep(3)
    print('in the test2',name,*args)

#test1=timer(test1)
test1()
test2('张三','23','男')

8. 装饰器传参

当有时候装饰器需要接收参数时，例如一个网站，用户登录home页的时候需要根据本地密码登录，登录到bbs的时候需要外部Idap验证时候，如何写一个用于登录的装饰器：

user,passwd='张三','123456'
def auth(auth_type):
    def outwrapper(func):
        def wrapper(*args,**kwargs):
            if auth_type == "loc":
                username=input("username:")
                password=input("password:")

                if user == username and passwd==password:
                    print("User has pass")
                    func(*args,**kwargs)
                    return "你好"
                else:
                    exit("Invalid username or password")
            elif auth_type =="ldap":
                print("搞毛线ldap,不会")
        return  wrapper
    return outwrapper
def index():
    print("welcome to index page")
'''
@auth(auth_type = "loc")可以分解为：
auth(auth_type = 'loc')
home=auth(home)
执行auth（auth_type = "loc）这时auth指向了outwrapper,auth_type='loc'
执行home=auth(home)时，auth(home)将home赋值给了funct同时执行了outwrapper()返回wrapper重新赋值给了home ,home指向wrapper，home()执行了wrapper()
'''
@auth(auth_type = "loc")
def home():
    print("welcome to home page")
@auth(auth_type = "ldap")
def bbs():
    print("welcome to bbs page")

index()
#home(1)
#print(home())#home已经指向了wrapper,因此应该设置wrapper的返回值
home()
bbs()

二、生成器

1. 特点

  1. 避免内存浪费
  2. 循环生成所需数据
  3. 生成器只有在调用时才会生成相应的数据
  4. 只记录当前位置（不能向前调用）
  5. 只有一个方法next()方法

列表生成式：

def func(i):
    i*=6
    return i
print([func(i) for i in range(10)])
print([i*2 for i in range(10)])
#改成()为创建了生成器
#调用才生成
b=(i*2 for i in range(10))
#通过__next__()来访问无法返回上一个，只能保存单前值
#一般用循环
print(b.__next__())
print(b.__next__())
print(b.__next__())
# for i in b:
#     print(i)

2. 实例分析：斐波那契生成器

def fib(max):
    n,a,b=0,0,1
    while n < max:
        #print(b)
        #yield b相当于挂起目前的操作，将b的值返回给生成器，等待再执行next操作时继续之后的操作
        yield b
        #相当于
        #t=(b,a+b)
        #a=t(0)
        #b=t(1)
        a,b = b,a+b
        n+=1
    return  'This is fib'#出异常后打印消息
#仅仅是创建了f生成器，并没有进入fib函数，也没有进行max=10的传参
f=fib(10)
while True:
    try:
        #调用了next(f)进入了生成器，待yield传回参数后赋值给x继续下一步操作
        x=next(f)
        print('f:',x)
    except StopIteration as e:
        #抓住bug并返回异常为生成器返回值
        print('Generator return value:',e.value)
        break

三、迭代器

可直接作用于for循环的数据类型有以下几种：

一类是集合数据类型，如list、tuple、dict、set、str等；

一类是generator，包括生成器和带yiel的generator function

这些可以直接作用于for循环的对象统称为可迭代对象：Iterable.

可以使用isinstance()判断一个对象是否是Iterable对象

#判断是否为可迭代对象
from collections import Iterable,Iterator
print(isinstance([],Iterable))
print(isinstance(set(),Iterable))
print(isinstance((),Iterable))
print(isinstance({},Iterable))
print(isinstance("你好",Iterable))
print(isinstance(100,Iterable))
print("==========end===========")
#判断是否为迭代器对象
print(isinstance((x for x in range(5)),Iterator))

将可迭代对象变成迭代器

#iter():将可迭代对象转变成迭代器
c=iter('abc')
print(c)
print(c.__next__())
print(c.__next__())
print(c.__next__())

四、json与pickle序列化

方法：dumps,dump,loads,load

dumps和dump，load和loads的区别：

dumps是将对象序列化

dump是将对象序列化并保存到文件中

loads将序列化字符串反序列化

load将序列化字符串从文件读取并反序列化

1. json序列化

#序列化：将内存中的内容挂起，存到硬盘上的过程
#多次dump时，会在文件中产生多条记录，导致不能load
#重点：每个程序只dump一次，想多次dump要dump成不同的文件，或者将源文件冲掉
import json
info={
    'name':'Alex',
    'age':23
}

f=open("test.txt","w",encoding='utf-8')
#print(json.dumps(info))
f.write(json.dumps(info))#与json.dump(info,f)相同
f.close()

2. json去序列化

适用于多语言的数据传输

import json
#反序列化：将硬盘上的内容重载到内存中
#f = open("test.txt",'r')
# data=f.read()
# f.close()
# print(data)
f=open("test.txt",'r')
data = json.loads(f.read())#与data=json.load(f)相同
print(data["age"])
